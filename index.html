1) Service Models in Cloud Computing

Cloud Services are divided into three main categories:

  • IaaS – Infrastructure as a Service
  • PaaS – Platform as a Service
  • SaaS – Software as a Service

IaaS (Infrastructure as a Service)

  • IaaS stands for Infrastructure as a Service
  • IaaS is a Virtual Infrastructure
  • It is Used by network architects and system admins
  • User controls:
    • Applications
    • Data
    • Runtime
    • Middleware
    • Operating System (OS)
  • Requires technical knowledge
  • Example: AWS (Amazon Web Services)

PaaS (Platform as a Service)

  • PaaS stands for Platform as a Service
  • PaaS is an app development platform
  • It is Used by software developers
  • User controls:
    • Applications
    • Data
  • Requires some technical knowledge
  • Example: Google App Engine

SaaS (Software as a Service)

  • SaaS stands for Software as a Service
  • It provides ready-to-use software over the internet
  • It is Used by end-users
  • User controls: Nothing
  • No technical knowledge required
  • Example: Gmail

2) Key Features and Services provided by AWS, Azure and Google Cloud

AWS:

  • AWS stands for Amazon Web Services
  • Key Features:
    • Cost Effectiveness:
      • Instead of buying external computer hardware or building data centers to store data, users can simply pay rent for storing the data in cloud servers.
      • If you use more, you pay more.
      • If you use less, you pay less.
    • Scalability:
      • The ability of a computing system to handle workloads by adjusting its performance and costs is called scalability.
      • Instead of upgrading the entire system, cloud systems allow you to add more servers to increase scalability.
    • Availability & Reliability:
      • Cloud systems store data at different locations around the world.
      • If one data center fails, the data is still safe in the cloud system, which makes it more reliable and highly available.
  • Cloud Services:
    • EC2
    • S3
    • EBS
    • SQS
    • SimpleDB

Microsoft Azure:

  • Key Features:
    • Works well with Microsoft tools
    • Provide communication between services
    • Load Balancing:
      • It distributes the workloads across multiple servers.
      • Load Balancing Algorithm is mainly of two types:
        • Static Load Balancing Algorithm
        • Dynamic Load Balancing Algorithm
  • Cloud Services:
    • Live Service
    • .NET Service
    • SharePoint Service
    • SQL Azure
    • Azure Tables

Google Cloud:

  • Key Features:
    • Can handle very large volumes of data
    • Scalability:
      • The ability of a computing system to handle workloads by adjusting its performance and costs is called scalability.
      • Instead of upgrading the entire system, cloud systems allow you to add more servers to increase scalability.
    • Availability & Reliability:
      • Cloud systems store data at different locations around the world.
      • If one data center fails, the data is still safe in the cloud system, which makes it more reliable and highly available.
  • Cloud Services:
    • Gmail
    • Google Docs
    • Google Search Engine
